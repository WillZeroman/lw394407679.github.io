写在开头的话

程序员，高大上的名字是工程师，接地气的名字是码农。作为码农中普普通通的一个，做着每天码砖的那些事，心里想着砌出万里长城，热爱技术，对新技术有强烈的好奇心。
技术博客，作为普通程序员向高级程序员进军的一个号角，不仅完美的诠释了开源共享的精神，而且平衡了程序员逻辑思维与语言表达能力。所以，新时代的码农们要用代码影响世界，更要用文字影响世界。
本文记录我搭建博客的过程，同时将经验分享给大家，相互帮助，相互学习。

相关技术

本博客是一个静态博客，这种静态博客适合于个人博客。采用 Hexo + Github Pages 搭建而成。
Hexo 是一个基于node.js的静态框架，其特点：简单、快速、强大。一条命令hexo init就搞定了。
GitHub Pages —— Websites for you and your projects. Hosted directly from your GitHub repository. Just edit, push, and your changes are live.
以上是官网的原话，简单来说就是一个免费的静态站点。
MarkDown 程序员写文档必备，语法简单。

起步

以下步骤是假定您已安装了node开发环境，对node有一定了解，熟悉git管理工具。本博客是在windows环境下安装的，所以配置node，git可能有点麻烦。

安装Hexo

  1. 全局安装hexo：

  2. 执行初始化

  3. 进入blogName目录，启动本地测试

好了，您可以在http://localhost:4000 访问所搭建的博客

创建GitHub Pages

  1. 登陆GItHub，创建一个repository，进入如下界面：

    Repository name 必须是: 自己用户名.github.io
    比如我的就是：lw394407679.github.io
    Description 任意写
    选择Public
    点击 Create repository
  2. 完成上述步骤后，我们已经创建了一个repository，现在点击Settings，找到GitHub Pages点击Launch automatic page generator，再点击Continue to layouts 任意选择一个布局，点击Publish page。好了，在浏览器中访问 http://用户名.github.io

Hexo与Github Pages结合

如何将之前利用Hexo搭建的本地博客发布到Gibhub上，需要通过如下操作：

  1. 修改blogName目录下_config.yml文件中的deploy选项：(ssh方式)

  2. 执行命令 npm install hexo-deployer-git --save 安装发布所需依赖

  3. 发布： 执行 hexo generate deploy 生成静态页面并发布到github上。然后再浏览器中访问 http://您的用户名.github.io

扩展

通过上面步骤，我们已经完成了Hexo + GitHub Pages的搭建，现在你可以美化你的博客和写文章了。

  • Hexo 提供了许多优美主题，我们只需要选择自己喜欢的就行。
  • 新建文章：可以使用命令 hexo new "postName" 创建一个文章，也可以直接在source/_posts目录下创建一个md文件。
  • 多说评论框：在多说开发者中心申请一个站点，具体部署参考Hexo使用多说教程

总结

博客的基本框架都搭建好了，现在开始非凡的技术积累之旅吧。
